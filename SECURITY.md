## 脆弱性の報告

セキュリティ脆弱性を発見した場合は、**公開のIssueで報告しないでください**。

代わりに、以下の方法で報告してください:

### 報告方法

1. **GitHubのSecurity Advisories**（推奨）
   - このリポジトリの「Security」タブから「Report a vulnerability」をクリック
   - 非公開で脆弱性を報告できます

2. **メール**
   - セキュリティ担当者にメールで連絡
   - メールアドレス: YOUR_EMAIL@example.com

### 報告に含める情報

脆弱性の報告には、以下の情報を含めてください:

- **脆弱性の種類** (例: SQL Injection, XSS, CSRF, etc.)
- **影響を受けるコンポーネント**
- **再現手順** (できるだけ詳細に)
- **PoC (Proof of Concept)** (可能であれば)
- **想定される影響**
- **提案する修正方法** (可能であれば)

### 対応プロセス

1. **受領確認**: 24時間以内に報告を受領したことを確認します
2. **初期評価**: 72時間以内に脆弱性の重大度を評価します
3. **修正計画**: 重大度に応じて修正スケジュールを策定します
4. **修正実装**: 可能な限り迅速に修正を実装します
5. **情報公開**: 修正版リリース後、適切なタイミングで脆弱性情報を公開します

### 重大度の分類

- **Critical**: 即座にシステムが侵害される可能性がある
- **High**: 重要な情報が漏洩する可能性がある
- **Medium**: 限定的な影響がある
- **Low**: 軽微な影響のみ

## セキュリティに関するベストプラクティス

このプロジェクトを使用する際は、以下のベストプラクティスに従ってください:

### APIキーとWebhook URLの管理

- **環境変数ファイル (.env) をGitリポジトリにコミットしない**
- APIキーやWebhook URLを公開しない
- `.gitignore` に `.env` が含まれていることを確認

### 依存関係の更新

- 定期的に依存パッケージを更新する
- 脆弱性が報告された依存関係は即座に更新

```bash
# 依存関係の脆弱性チェック
pip install safety
safety check
```

### 権限の最小化

- YouTube APIキーには必要最小限の権限のみを付与
- Discord Webhookは特定のチャンネルのみにスコープ

### ログの適切な管理

- APIキーやトークンがログに記録されないよう注意
- ログファイルを適切に保護（権限設定）

### 本番環境での運用

- Pythonとすべての依存関係を最新の安定版に保つ
- システムログを定期的に監視
- 異常なAPI使用パターンを検知

## 既知の制限事項

### YouTube Data API v3のクォータ制限

- 無料枠は1日10,000クォータ
- クォータ超過時は監視が停止します
- 悪意ある第三者によるクォータ消費攻撃のリスクがあります

**対策**:
- `check_interval` を適切に設定（推奨: 300秒以上）
- Google Cloud ConsoleでAPIキーの使用状況を監視

### Discord Webhook URLの取り扱い

- Webhook URLが漏洩すると、任意のメッセージが投稿可能になります

**対策**:
- Webhook URLを絶対に公開しない
- 定期的にWebhook URLを再生成
